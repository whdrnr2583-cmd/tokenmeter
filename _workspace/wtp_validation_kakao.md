# 카톡 maker-share — Token Meter 공유 + 신호 수집

> **D-039** (개정 2026-05-19). D-031 단발 우회 · 카톡 1채널.
> 방식 = "내가 만든 도구를 공유" + 답글에서 대화. 영업 아님 — 만든 사람이 나누는 것.
> 누가 필요하면 가져다 쓰게. 1차 목표 = free 도구에 pull(관심)이 있는지 + 통증 가진 사람 찾기.

## 원칙

1. **opener에 Pro·$5·가격 페이지 절대 안 꺼냄.** 무료 도구를 공유하는 것뿐.
2. **한 번만 자연스럽게** — 도배 X. 진짜 정보는 *답글 대화*에서 나온다.
3. 링크는 **npm(무료 OSS)**, 가격 랜딩 X.
4. **올리고 끝 = broadcast.** 반응한 사람과 대화해야 신호 수집. 이게 핵심.
5. 강요 X — 미지근하면 그냥 넘어간다.

## opener (카톡 — 캡쳐 + "받아서 쓰세요" 형태)

**1. Claude Code 스크린샷 1장**
Claude Code 안에서 토큰 사용량을 물어본 화면을 캡쳐. 가장 깔끔한 건 `/token-meter`
슬래시 명령 결과 (오늘 사용량 + MCP·도구별 한 화면).
- 슬래시 없으면: `npx -y @whdrnr2583/token-meter install-command claude-code`
  → Claude Code 재시작
- 또는 그냥 "token-meter로 이번주 사용량 보여줘"라고 물어본 결과를 캡쳐
⚠️ **출력에 실제 프로젝트 폴더명이 나옴** — 민감한 프로젝트명은 가리고 올릴 것.
금액은 둬도 됨(도구가 뭘 보여주는지의 증거).

**2. 스크린샷 밑 짧은 글 (2줄):**

> Claude Code에서 토큰 얼마 썼는지 물어보면 답해주는 MCP 만들었어요 ㅎㅎ 무료요.
> `npx @whdrnr2583/token-meter install-mcp claude-code` 치면 바로 붙어요.

길게 X — 캡쳐가 설명을 대신한다. 글은 2줄.
opener에 Pro·$5 비노출 (무료 도구 공유). 가격은 누가 물으면 답글에서 한 줄.

## 누가 반응하면 — 답글에서 (자연스럽게, 심문 아님)

- 지금 토큰 비용은 어떻게 확인하세요?

   ("기본은 무료고, 30일 히스토리·캐시 효율 같은 건 월 $5 Pro예요"). 먼저 안 꺼냄.

## 신호 해석

| 강(强) | 중(中) | 약(弱) |
|---|---|---|
| 설치 후 피드백 가져옴 / "이거 딱 필요했어요" / 기능 콕 집어 질문 / 먼저 Pro 물어봄 | 비용 통증을 구체적으로 얘기함 / ccusage 등 이미 써봄 | 👍 이모지만 / 무반응 / "오 좋네요"만 |

## 이 라운드가 답하는 것 / 안 하는 것

- **답함**: free 도구에 pull이 있나? 누가 토큰 비용 통증을 갖고 있나?
- **아직 안 함**: Pro $5 WTP — 그건 free가 먼저 당겨진 뒤의 질문.
- 순서가 맞다: 아무도 free를 안 원하면 Pro는 의미 없음.
